# Regra: Atualização Automática de Documentação
Sempre que houver alterações na lógica do projeto, estrutura do banco de dados ou novas funcionalidades:
1. Analise o impacto das mudanças nos arquivos: `doc.md`, `schema.md` e `todo.md`.
2. Atualize o conteúdo desses arquivos para refletir o estado atual do projeto.
3. Não finalize a tarefa sem garantir que a documentação está sincronizada com o código.
